## 文档维护规则

1. **改产品语义** → 先改 `PHASE4-PLAN-*.md` §任务定义，再传播
2. **加新功能** → 在 `PHASE4-PLAN-*.md` 加 V-XX 条目
3. **完成阶段** → 标 `[x]` + 完成时间，并在 `REFACTOR-LOG.md` 追加条目
4. **会话结束** → 新建 `SESSION-SUMMARY-YYYY-MM-DD.md`（按执行日期命名）
5. **过期文档** → 移到 `archive/` 子目录（如 `2026-06-06-plans/`），加 ⚠️ 头注，**不直接删除**
6. **handover 等中间段文档**（VP/PM 给工程师的过渡文档）→ 评估：保留价值（避坑点、决策依据、设计意图）合并到 `docs/` 根对应文档；过期失效的**直接删除**
