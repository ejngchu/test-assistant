## 3. 详细任务分解（V-01..V-10）

### V-01 · 错题权重系统

**是什么**：每道错题有一个 `weight`（数字，越高 = 越弱），影响复习优先级和练习生成。

**为什么**：用户原话："录入的错题有个权重系统，主要维持错题权重，针对知识盲区练习"。

**数据模型**（`server/src/db/schema.ts`）：
```ts
mistakes: {
  // ... 现有字段
  weight: integer('weight').notNull().default(100), // 100 = 新录入；越低 = 越掌握
  lastWeightUpdate: text('last_weight_update'),
}
```

**weight 变化规则**：

| 操作 | weight 变化 |
|------|------------|
| 录入新错题 | = 100（默认）|
| 复习答"不会" | + 30（最高 200）|
| 复习答"犹豫" | - 10 |
| 复习答"掌握" | - 50（最低 10）|
| 答对练习题（关联 knowledgePoint）| 该 kp 下的错题 - 5 |
| 答错练习题 | + 15 |

**API**：
| Method | Path | 用途 |
|--------|------|------|
| GET | `/api/study/mistake/:id` | 单条详情（已有，扩展） |
| PATCH | `/api/study/mistake/:id/weight` | 手动改权重 |
| GET | `/api/study/mistake/weak?limit=N` | 取权重最高的 N 道错题 |

**验收**：
- [ ] 录入错题 weight = 100
- [ ] 答"掌握"后 weight 减少
- [ ] 答"不会"后 weight 增加
- [ ] 列表按 weight 降序展示
- [ ] 单元测试：所有规则

### V-02 · 错题复习等级 UI

**是什么**：错题详情页改为"像背单词一样"的三档复习。

**为什么**：用户原话："点进详情，只显示题目，不显示答案，像背单词一样给用户三个选项不会、会了、掌握三个等级"。

**三档定义**：

| 等级 | 图标 | 含义 | 触发的 weight 变化 |
|------|------|------|--------------------|
| 不会 | 😟 / 红色 X | 完全没思路 | + 30 |
| 犹豫 | 🤔 / 黄色 ！ | 大致会但不确定 | - 10 |
| 掌握 | 😎 / 绿色 ✓ | 完全掌握 | - 50 |

**UI 流程**：
1. 进入错题详情 → 只显示**题目图片** + 答题选项（不显示答案/解析）
2. 三个按钮横向铺底（不会 / 犹豫 / 掌握）
3. 点击任一等级 → 弹出答案 + 解析（蒙层 or 下方卡片）
4. 弹出"已标记为 X"toast
5. 返回错题本（自动按新 weight 重排）

**修改文件**：
- `src/pages/mistake-detail/index.tsx` —— 重做详情页布局
- `src/store/appStore.ts` —— 加 `weight` 字段、`updateWeight` action
- `server/src/study/study.service.ts` —— `markReview` 接受 `level` 参数

**验收**：
- [ ] 详情页只显示题目不显示答案
- [ ] 三个按钮横向铺底
- [ ] 点"不会"显示答案+解析 + weight +30
- [ ] 点"掌握" weight -50
- [ ] weight 变化持久化到 DB

### V-03 · 练习生成系统（我的页面入口）

**是什么**："我的"页面入口 → 选择条件 → 生成练习题 → 可打印 → 答题 → 提交。

**为什么**：用户原话："我的页面能有一个练习生成系统，根据知识盲区生成针对性的考题，考题科目、数量有默认值，也可以自行调整，考题可打印"。

**默认 vs 可调**：
- **科目**：默认"全部"（chinese/math/english），可单选/多选
- **数量**：默认 10 道，可调 5/10/15/20
- **知识点来源**：默认"按 weight 取 top N 错题的 kp"
- **难度**：默认混合（easy:medium:hard = 4:4:2），可调

**UI 流程**：
1. "我的"页面加"📝 练习中心"入口（独立大按钮）
2. 进入 → 选条件页（表单：科目/数量/难度）
3. 提交 → 调 `POST /api/study/practice/generate`
4. 答题页（已有 `practice-answer`）
5. 答完 → 显示结果
6. **打印**（仅 H5）：调 `window.print()` + CSS `@media print` 隐藏非题目元素

**API 扩展**（`POST /api/study/practice/generate`）：
```ts
{
  userId: string,
  subject?: 'chinese' | 'math' | 'english' | 'all',  // 新增
  count: number,                                       // 已存在
  knowledgePoints?: string[],                          // 已存在
  difficulty?: 'easy' | 'medium' | 'hard' | 'mixed',  // 新增
  source?: 'weak_points' | 'manual',                   // 新增
}
```

**验收**：
- [ ] 我的页面有"练习中心"入口
- [ ] 表单默认值合理
- [ ] 生成题量准确
- [ ] 答题流程跑通

### V-06 · 公式 + 图形渲染

**是什么**：数学题中的公式与图形在错题详情中正确显示。

**问题**：
- 现有 `<Image>` 直接显示拍照图，公式/图形本身是图片 → 没问题
- LLM OCR 后的 `correctAnswer` / `analysis` 文本中可能含 `\( \)` `\[ \]` `$$` 等 LaTeX 语法
- 当前用 `<Text>` 直接显示 → 渲染为字面量

**方案选型**：

| 方案 | 优劣势 |
|------|--------|
| **KaTeX** | 快速、轻量；纯函数式渲染；广泛用于 math.stackexchange |
| MathJax | 完整 TeX 支持；体积大（~2MB） |
| **图片替换**（最稳）| 录入时 LLM 把 LaTeX 转图片，存储 `mathjax-rendered.png`；前端直接 `<Image>` |

**推荐**：先 **KaTeX** + 文本 inline 显示；后续若 LLM 排版复杂再切图片方案。

**实现位置**：
- `src/components/ui/math-text.tsx` —— 包装 KaTeX，识别 `\(...\)` `\[...\]` `$...$` 切分
- `mistake-detail`、`homework-result`、`practice-answer` 的 `correctAnswer` / `analysis` / `hint` 用 `<MathText>` 替代 `<Text>`

**验收**：
- [ ] KaTeX 接入
- [ ] `\(a^2 + b^2 = c^2\)` 正确渲染为 a²+b²=c²
- [ ] 公式 + 图形（图形是 Image）在同一卡片中正确显示
