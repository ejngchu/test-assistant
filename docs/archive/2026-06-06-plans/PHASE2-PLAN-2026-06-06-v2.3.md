## 2. 详细任务分解

### 2.1 切换 LLM 到 Provider 1（MiniMax-M3）（P0）

**目标**：`llm.service.ts` 验证 Provider 1（默认）真实可用，Settings 关闭"演示模式"后能调通

**任务列表**：
- [ ] 2.1.1 确认 `server/.env.local` 已配 3 个 Provider（用户已提供 key，配置应已就位）
      ```
      LLM_PROVIDER=openai
      OPENAI_API_KEY=<Provider 1 key>
      OPENAI_BASE_URL=https://minnimax.chat/v1
      OPENAI_MODEL=MiniMax-M3
      ```
- [ ] 2.1.2 跑 `server/scripts/test-llm.mjs` 验证文本 + vision 调用
- [ ] 2.1.3 重启后端 → Settings 关掉"演示模式" → 拍一道真错题 → 验证非 mock 模式工作
- [ ] 2.1.4 如果 Provider 1 不稳定，临时切到 Provider 2（通义千问）或 Provider 3（DeepSeek V4）
- [ ] 2.1.5 单元测试：Mock fetch/axios，验证请求格式和响应解析

**验收标准**：
- ✅ 拍一道真错题 → 5 秒内得到真实 AI 分析（不是 mock 固定文本）
- ✅ `pnpm --filter server test` 通过
- ✅ Settings 开关切换 mock/real 实时生效
- ✅ Provider 切换在 `.env.local` 改 3 个变量 + 重启即可

**回退方案**：
- 切 Provider 2：`OPENAI_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1`，`OPENAI_MODEL=qwen3.7-plus`
- 切 Provider 3：`OPENAI_BASE_URL=https://tokenhub.tencentmaas.com/v1/chat/completions`，`OPENAI_MODEL=deepseek-v4-pro-202606`
- 备选（用户外的免费方案）：智谱 GLM-4-Flash（`https://open.bigmodel.cn/api/paas/v4/`，`glm-4v-flash`）或 SiliconFlow（`https://api.siliconflow.cn/v1/`）
